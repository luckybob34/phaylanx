# RFP Response Template

> Section structure reference for RFP responses. The proposal-writer agent uses this to maintain consistent format.

---

## Cover Letter

```markdown
[Date]

[Addressee Name]
[Title]
[Organization]
[Address]

Re: [RFP Title / Number]

Dear [Name],

[Opening - express interest, reference RFP by name and number]

[Value proposition - 2-3 sentences on why you're the right choice. Lead with client priorities.]

[Differentiators - specific metrics, certifications, relevant experience.]

[Closing - confirm compliance, note exceptions, express availability.]

Sincerely,
[Name, Title]
[Company]
[Contact Information]
```

---

## Executive Summary

- **Understanding of Need** — Restate client's problem. Show you understand WHAT and WHY.
- **Our Approach** — High-level solution, 3-5 sentences. Technical but accessible.
- **Why Us** — 2-3 differentiators with evidence (metrics, past performance, unique capabilities).
- **Commitment** — Timeline, quality, partnership. Name the project lead.

---

## Technical Approach

- **Current State Assessment** — What exists today
- **Proposed Solution** — Architecture overview, diagrams
- **Implementation Methodology** — Phases with descriptions
- **Technology Stack** — Component / Technology / Rationale table
- **Security and Compliance** — Standards addressed (SOC 2, FedRAMP, HIPAA)
- **Integration Approach** — Connections to existing systems

---

## Management Approach

- **Project Governance** — Communication cadence, reporting, escalation
- **Team Structure** — Role / Name / Qualifications / Allocation table
- **Quality Assurance** — Review processes, testing methodology
- **Risk Management** — Top 3-5 risks with mitigation strategies

---

## Past Performance

For each reference:
- **Client** / **Scope** / **Relevance** / **Outcome** (quantified) / **Reference** (if permitted)

---

## Cost Proposal

- **Pricing Summary** — Phase / Description / Cost table
- **Assumptions** — What the pricing depends on
- **Payment Schedule** — Milestone / Trigger / Amount table
- **Rate Card** (if T&M) — Role / Hourly Rate table

---

## Formatting Rules

1. Mirror RFP numbering
2. One requirement, one response
3. Bold the requirement first, then respond beneath
4. Use tables for comparisons
5. Include cross-references
6. Stay within page limits
7. Every section in ToC must exist
